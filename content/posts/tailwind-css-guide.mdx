Tailwind CSS는 유틸리티 퍼스트 CSS 프레임워크입니다. 미리 정의된 클래스를 조합해서 스타일을 적용하는 방식으로, 별도의 CSS 파일을 작성할 필요 없이 HTML 안에서 직접 디자인할 수 있습니다. 처음에는 클래스가 길어 보여 거부감이 있을 수 있지만, 한번 익숙해지면 기존 CSS보다 훨씬 빠르게 UI를 만들 수 있습니다.

## 왜 Tailwind CSS인가?

### 기존 CSS의 문제점

전통적인 CSS 개발 방식에는 몇 가지 고질적인 문제가 있습니다.

- **클래스 이름 짓기의 고통**: `.card-wrapper`, `.card-container`, `.card-content` 등 이름을 짓는 데 시간이 소모됩니다. BEM 같은 방법론도 근본적인 해결은 아닙니다.
- **CSS 파일의 비대화**: 프로젝트가 커질수록 CSS 파일이 무한히 증가합니다. 사용하지 않는 스타일이 남아있어도 삭제하기 두렵습니다.
- **컨텍스트 전환**: HTML을 수정하려면 CSS 파일을 함께 열어야 합니다. 파일 사이를 오가는 것은 생산성을 떨어뜨립니다.
- **전역 스코프**: CSS는 기본적으로 전역이므로 다른 컴포넌트에 영향을 줄 수 있습니다.

### Tailwind의 해결 방법

Tailwind는 미리 정의된 유틸리티 클래스를 조합하는 방식으로 이 문제들을 해결합니다.

```html
<!-- 전통 CSS -->
<div class="card">
  <h2 class="card-title">제목</h2>
  <p class="card-description">내용</p>
</div>

<!-- Tailwind CSS -->
<div class="rounded-lg border p-6 shadow-md">
  <h2 class="mb-2 text-xl font-bold">제목</h2>
  <p class="text-gray-600">내용</p>
</div>
```

별도의 CSS 파일이 필요 없습니다. 클래스 이름을 고민할 필요도 없습니다. HTML만 보면 어떤 스타일이 적용되어 있는지 바로 알 수 있습니다.

## 핵심 유틸리티 클래스

### 레이아웃

```html
<!-- Flexbox -->
<div class="flex items-center justify-between gap-4">
  <div>왼쪽</div>
  <div>오른쪽</div>
</div>

<!-- Grid -->
<div class="grid grid-cols-1 gap-6 sm:grid-cols-2 lg:grid-cols-3">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</div>
```

### 스페이싱

Tailwind의 스페이싱 스케일은 4px 단위입니다. `p-1`은 4px, `p-4`는 16px, `p-8`은 32px입니다.

- `p-{n}`: padding 전체
- `px-{n}`: padding 좌우, `py-{n}`: padding 상하
- `m-{n}`: margin 전체
- `mx-auto`: 가로 중앙 정렬
- `gap-{n}`: flex/grid 간격

### 타이포그래피

```html
<h1 class="text-3xl font-bold text-gray-900">큰 제목</h1>
<p class="text-base text-gray-600 leading-relaxed">
  본문 텍스트입니다. leading-relaxed로 줄 간격을 넓혀 가독성을 높입니다.
</p>
<span class="text-sm text-gray-500">작은 보조 텍스트</span>
```

### 색상 시스템

Tailwind는 체계적인 색상 팔레트를 제공합니다. 각 색상은 50(가장 밝음)부터 950(가장 어두움)까지의 단계를 가집니다.

```html
<div class="bg-blue-500 text-white">파란 배경</div>
<div class="text-gray-700">짙은 회색 텍스트</div>
<div class="border-gray-200">밝은 회색 테두리</div>
```

## 반응형 디자인

Tailwind의 반응형 디자인은 모바일 퍼스트 접근법을 사용합니다. 접두어 없는 클래스가 모바일에 적용되고, 브레이크포인트 접두어로 큰 화면의 스타일을 추가합니다.

```html
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4">
  <!-- 모바일: 1열, sm(640px): 2열, lg(1024px): 3열, xl(1280px): 4열 -->
</div>
```

### 브레이크포인트

- `sm`: 640px 이상
- `md`: 768px 이상
- `lg`: 1024px 이상
- `xl`: 1280px 이상
- `2xl`: 1536px 이상

## 다크 모드

Tailwind는 `dark:` 접두어로 다크 모드 스타일을 쉽게 추가할 수 있습니다.

```html
<div class="bg-white dark:bg-gray-900">
  <h1 class="text-gray-900 dark:text-white">제목</h1>
  <p class="text-gray-600 dark:text-gray-400">내용</p>
</div>
```

`tailwind.config.js`에서 `darkMode: 'class'`로 설정하면 JavaScript로 다크 모드를 토글할 수 있습니다.

## 실전 컴포넌트 예시

### 카드 컴포넌트

```html
<div class="overflow-hidden rounded-xl border border-gray-200 bg-white
            shadow-sm transition-shadow hover:shadow-lg
            dark:border-gray-800 dark:bg-gray-900">
  <img class="h-48 w-full object-cover" src="..." alt="..." />
  <div class="p-6">
    <span class="rounded-full bg-blue-100 px-3 py-1 text-xs font-medium
                 text-blue-700 dark:bg-blue-900 dark:text-blue-300">
      카테고리
    </span>
    <h3 class="mt-3 text-lg font-semibold text-gray-900 dark:text-white">
      카드 제목
    </h3>
    <p class="mt-2 text-sm text-gray-600 dark:text-gray-400">
      카드 설명 텍스트가 여기에 들어갑니다.
    </p>
  </div>
</div>
```

### 버튼 컴포넌트

```html
<!-- 기본 버튼 -->
<button class="rounded-lg bg-blue-600 px-4 py-2 text-sm font-medium
               text-white hover:bg-blue-700 focus:outline-none
               focus:ring-2 focus:ring-blue-500 focus:ring-offset-2
               disabled:opacity-50 disabled:cursor-not-allowed">
  버튼
</button>

<!-- 아웃라인 버튼 -->
<button class="rounded-lg border border-gray-300 px-4 py-2 text-sm
               font-medium text-gray-700 hover:bg-gray-50
               dark:border-gray-700 dark:text-gray-300
               dark:hover:bg-gray-800">
  아웃라인
</button>
```

## Tailwind CSS v4의 변화

2026년 기준 Tailwind CSS v4가 출시되면서 중요한 변화가 있었습니다.

- **Oxide 엔진**: Rust 기반의 새 엔진으로 빌드 속도가 대폭 개선
- **CSS-first 설정**: `tailwind.config.js` 대신 CSS 파일에서 직접 설정 가능
- **네이티브 CSS 중첩**: PostCSS 없이도 중첩 선택자 사용 가능
- **자동 콘텐츠 감지**: `content` 배열 설정이 불필요

## 생산성 팁

### VS Code 확장 프로그램

**Tailwind CSS IntelliSense**는 필수 확장 프로그램입니다. 클래스명 자동완성, 미리보기, 린팅을 제공합니다.

### Prettier 플러그인

`prettier-plugin-tailwindcss`를 설치하면 클래스 순서를 자동으로 정렬해줍니다. 팀 내에서 일관된 스타일을 유지할 수 있습니다.

### cn() 유틸리티 함수

조건부 클래스를 깔끔하게 관리할 수 있는 유틸리티입니다.

```typescript
import { clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

// 사용
<button className={cn(
  "rounded-lg px-4 py-2",
  isActive && "bg-blue-600 text-white",
  isDisabled && "opacity-50 cursor-not-allowed"
)}>
```

## 마무리

Tailwind CSS는 "유틸리티 클래스가 너무 길다"는 초기 거부감만 넘기면 가장 생산적인 CSS 도구입니다. 디자인 시스템의 일관성을 유지하면서도 빠르게 UI를 구현할 수 있고, 사용하지 않는 CSS가 자동으로 제거되므로 번들 크기도 최적화됩니다.

React, Vue, Svelte 등 어떤 프레임워크와도 잘 어울리며, Next.js와 조합하면 서버 컴포넌트에서도 문제없이 동작합니다. 아직 사용해보지 않았다면, 작은 프로젝트에서 한번 시도해보세요.
