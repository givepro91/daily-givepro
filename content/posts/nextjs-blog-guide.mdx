블로그 플랫폼을 사용하면 간편하지만, 플랫폼 수수료와 제한된 커스터마이징 때문에 수익화에 한계가 있습니다. Next.js로 직접 블로그를 만들면 100% 수익을 가져가면서도 원하는 대로 디자인하고 기능을 추가할 수 있습니다.

이 가이드에서는 프로젝트 생성부터 배포, 수익화 설정까지 전체 과정을 단계별로 설명합니다. 코딩 경험이 조금이라도 있다면 충분히 따라할 수 있습니다.

## 왜 Next.js인가?

블로그를 만들 수 있는 도구는 많습니다. WordPress, Ghost, Hugo, Gatsby 등 다양한 선택지가 있는데, Next.js를 선택한 이유는 다음과 같습니다.

### SEO 최적화가 뛰어납니다

Next.js는 서버 사이드 렌더링(SSR)과 정적 사이트 생성(SSG)을 모두 지원합니다. 블로그 글은 빌드 시점에 HTML로 미리 생성되기 때문에 검색 엔진 크롤러가 콘텐츠를 완벽하게 인덱싱할 수 있습니다. 메타데이터 API도 내장되어 있어 OpenGraph 태그, JSON-LD 구조화 데이터를 쉽게 추가할 수 있습니다.

### 성능이 빠릅니다

정적으로 생성된 페이지는 CDN에서 직접 서빙되므로 서버 응답 시간이 매우 짧습니다. 이미지 최적화, 폰트 최적화, 코드 스플리팅 등 성능 최적화 기능이 기본으로 내장되어 있습니다. Google의 Core Web Vitals 점수가 높아질수록 검색 순위에도 긍정적인 영향을 미칩니다.

### 무료로 배포할 수 있습니다

Vercel에서 개인 프로젝트를 무료로 호스팅할 수 있습니다. GitHub에 코드를 push하면 자동으로 빌드되고 배포됩니다. SSL 인증서, CDN, 자동 배포까지 모두 무료입니다. 커스텀 도메인도 연결할 수 있습니다.

### React 생태계의 장점

React 컴포넌트를 활용하면 광고 배너, 제품 카드, 뉴스레터 폼 등 수익화에 필요한 요소들을 컴포넌트로 만들어 재사용할 수 있습니다. MDX를 사용하면 마크다운 안에서 이런 컴포넌트를 바로 사용할 수 있어 콘텐츠 작성이 매우 유연해집니다.

## 프로젝트 시작하기

### 1단계: 프로젝트 생성

터미널에서 다음 명령어로 Next.js 프로젝트를 생성합니다.

```bash
npx create-next-app@latest my-blog --typescript --tailwind --app --src-dir
```

옵션 설명:
- `--typescript`: 타입 안전성을 위해 TypeScript를 사용합니다
- `--tailwind`: 빠른 스타일링을 위해 Tailwind CSS를 설정합니다
- `--app`: 최신 App Router를 사용합니다
- `--src-dir`: 소스 코드를 src 디렉토리에 정리합니다

### 2단계: 프로젝트 구조 설계

블로그에 필요한 폴더 구조를 만듭니다.

```
my-blog/
├── content/
│   └── posts/          # MDX 블로그 글
├── src/
│   ├── app/
│   │   ├── blog/
│   │   │   ├── [slug]/
│   │   │   │   └── page.tsx    # 개별 글 페이지
│   │   │   └── page.tsx        # 글 목록
│   │   ├── page.tsx            # 홈페이지
│   │   └── layout.tsx          # 공통 레이아웃
│   ├── components/
│   │   ├── ads/        # 광고 컴포넌트
│   │   ├── blog/       # 블로그 관련 컴포넌트
│   │   └── layout/     # 레이아웃 컴포넌트
│   └── lib/
│       ├── posts.ts    # 글 데이터 처리
│       └── types.ts    # TypeScript 타입 정의
└── public/
    └── images/         # 이미지 파일
```

### 3단계: MDX 설정

MDX는 마크다운 안에서 JSX 컴포넌트를 사용할 수 있게 해주는 형식입니다. 필요한 패키지를 설치합니다.

```bash
npm install next-mdx-remote gray-matter reading-time
npm install remark-gfm rehype-slug rehype-autolink-headings rehype-prism-plus
```

## 마크다운으로 블로그 글 작성하기

### Frontmatter 활용

각 MDX 파일 상단에 메타데이터를 YAML 형식으로 작성합니다.

```markdown
---
title: "글 제목"
description: "SEO에 사용될 설명"
date: "2026-02-19"
category: "tutorial"
tags: ["Next.js", "블로그"]
thumbnail: "/images/posts/my-post.jpg"
enableAds: true
---

글 내용을 마크다운으로 작성합니다...
```

이 메타데이터는 글 목록, SEO 메타태그, OG 이미지 등에 자동으로 활용됩니다. `gray-matter` 라이브러리가 파싱을 담당합니다.

### 글 데이터 처리 함수

블로그 글을 파일 시스템에서 읽어오는 유틸리티 함수를 만듭니다.

```typescript
import fs from "fs";
import path from "path";
import matter from "gray-matter";
import readingTime from "reading-time";

const postsDir = path.join(process.cwd(), "content/posts");

export function getAllPosts() {
  const files = fs.readdirSync(postsDir).filter(f => f.endsWith(".mdx"));
  return files
    .map(file => {
      const content = fs.readFileSync(path.join(postsDir, file), "utf8");
      const { data, content: body } = matter(content);
      return {
        slug: file.replace(/\.mdx$/, ""),
        ...data,
        readingTime: Math.ceil(readingTime(body).minutes),
        content: body,
      };
    })
    .filter(post => post.published !== false)
    .sort((a, b) => (a.date > b.date ? -1 : 1));
}
```

## SEO 최적화 설정

검색 엔진에 잘 노출되려면 기술적 SEO를 꼼꼼하게 설정해야 합니다.

### 메타데이터 API 활용

Next.js의 Metadata API를 사용하면 각 페이지별로 SEO 메타태그를 설정할 수 있습니다.

```typescript
export async function generateMetadata({ params }): Promise<Metadata> {
  const post = getPostBySlug(params.slug);
  return {
    title: post.title,
    description: post.description,
    openGraph: {
      title: post.title,
      description: post.description,
      type: "article",
      publishedTime: post.date,
    },
    alternates: {
      canonical: `https://my-blog.com/blog/${params.slug}`,
    },
  };
}
```

### JSON-LD 구조화 데이터

구조화 데이터를 추가하면 검색 결과에서 리치 스니펫(별점, 작성자, 날짜 등)이 표시될 수 있습니다.

```typescript
const jsonLd = {
  "@context": "https://schema.org",
  "@type": "Article",
  headline: post.title,
  datePublished: post.date,
  author: { "@type": "Person", name: "블로그 작성자" },
};
```

### 사이트맵과 robots.txt

Next.js의 내장 기능으로 사이트맵과 robots.txt를 자동 생성할 수 있습니다. `app/sitemap.ts`와 `app/robots.ts` 파일을 만들면 됩니다. 사이트맵에는 모든 글의 URL, 최종 수정일, 우선순위를 포함시킵니다.

### RSS 피드

RSS 피드를 제공하면 RSS 리더 사용자를 추가로 확보할 수 있습니다. `app/feed.xml/route.ts`에서 XML 형식의 RSS 2.0 피드를 생성합니다.

## 수익화 전략

블로그 수익화에는 크게 세 가지 방법이 있습니다.

### 1. Google AdSense - 광고 수익

Google AdSense는 가장 기본적인 블로그 수익화 방법입니다. 승인을 받으려면 다음 조건을 충족해야 합니다.

- **양질의 콘텐츠 15-20개 이상**: 각 글이 1,000단어 이상의 유용한 내용이어야 합니다
- **개인정보처리방침 페이지**: 필수 요건입니다
- **사이트맵 제출**: Google Search Console에서 사이트맵을 제출해야 합니다
- **고유한 콘텐츠**: 다른 사이트에서 복사한 내용이 아닌 오리지널 콘텐츠여야 합니다

AdSense가 승인되면 글 상단, 중간, 하단에 광고를 배치할 수 있습니다. 자연스러운 위치에 광고를 넣는 것이 클릭률을 높이는 핵심입니다.

### 2. 제휴 마케팅 - 제품 추천 수익

블로그 글과 관련된 제품을 소개하고 구매가 이루어지면 수수료를 받는 방식입니다. 쿠팡 파트너스, 아마존 어소시에이트 등의 프로그램에 가입할 수 있습니다.

제휴 마케팅의 핵심 원칙:
- 직접 사용해본 제품만 추천할 것
- 장점뿐 아니라 단점도 솔직하게 공유할 것
- 제휴 링크임을 투명하게 밝힐 것 (FTC 가이드라인)
- `rel="nofollow sponsored"` 속성을 반드시 추가할 것

### 3. 스폰서 콘텐츠 - 기업 협찬

블로그 트래픽이 일정 수준 이상이 되면 기업에서 직접 콘텐츠 제작을 의뢰하기도 합니다. 이때는 글 상단에 스폰서 콘텐츠임을 명시해야 합니다.

## Vercel에 배포하기

### 1단계: GitHub 저장소 연결

코드를 GitHub에 push한 후 Vercel 대시보드에서 저장소를 연결합니다.

### 2단계: 환경 변수 설정

Vercel 대시보드의 Settings > Environment Variables에서 필요한 환경 변수를 설정합니다.

```
NEXT_PUBLIC_SITE_URL=https://my-blog.com
NEXT_PUBLIC_ADSENSE_CLIENT_ID=ca-pub-XXXXXXXXXX
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
```

### 3단계: 커스텀 도메인 연결

Vercel에서 제공하는 `.vercel.app` 도메인 대신 자체 도메인을 연결하면 브랜딩에 유리합니다. 도메인 등록 업체에서 DNS를 Vercel로 설정하면 됩니다.

## 콘텐츠 전략 팁

기술적으로 완벽한 블로그를 만들어도 콘텐츠가 없으면 의미가 없습니다.

- **키워드 리서치**: Google Keyword Planner나 Ubersuggest를 활용해 검색량이 있는 키워드를 찾으세요
- **정기적 발행**: 주 1-2회 꾸준히 글을 발행하는 것이 중요합니다
- **긴 형식 콘텐츠**: 1,500-3,000단어의 심층 콘텐츠가 검색 순위에 유리합니다
- **업데이트**: 오래된 글을 최신 정보로 업데이트하면 검색 순위가 회복됩니다

## 마무리

Next.js로 수익화 블로그를 만드는 것은 초기 투자(시간과 노력)가 필요하지만, 장기적으로 보면 플랫폼 의존 없이 지속 가능한 수익 채널을 만드는 가장 좋은 방법입니다.

가장 중요한 것은 시작하는 것입니다. 완벽한 블로그를 만들려고 하지 말고, 최소한의 기능으로 먼저 런칭한 후 글을 쓰면서 점진적으로 개선하세요. 블로그의 가치는 기술 스택이 아니라 콘텐츠에서 나옵니다.
