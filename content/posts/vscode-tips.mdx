VS Code는 전 세계에서 가장 많이 사용되는 코드 에디터입니다. Stack Overflow의 2025년 설문조사에 따르면 개발자의 약 73%가 VS Code를 주력 에디터로 사용하고 있습니다. 하지만 기본 설정만으로 사용하고 있다면 이 강력한 도구의 절반도 활용하지 못하는 것과 같습니다.

이 글에서는 VS Code의 생산성을 극대화할 수 있는 단축키, 확장 프로그램, 설정 팁을 체계적으로 정리합니다. 당장 내일부터 적용할 수 있는 실전 팁들이니 꼭 따라해 보세요.

## 꼭 알아야 할 핵심 단축키

단축키를 외우는 것만으로도 개발 속도가 눈에 띄게 빨라집니다. 아래 단축키는 Mac 기준이며, Windows에서는 `Cmd`을 `Ctrl`로 바꾸면 됩니다.

### 파일 탐색과 이동

파일 사이를 빠르게 이동하는 것은 생산성의 기본입니다.

- **`Cmd + P`**: 빠른 파일 열기. 프로젝트 내 모든 파일을 이름으로 검색할 수 있습니다. 파일명의 일부만 입력해도 퍼지 매칭으로 찾아줍니다.
- **`Cmd + Shift + P`**: 명령 팔레트. VS Code의 모든 명령에 접근할 수 있는 만능 도구입니다. 설정 변경, 확장 프로그램 실행, 테마 변경 등 거의 모든 작업이 가능합니다.
- **`Cmd + B`**: 사이드바 토글. 코딩에 집중할 때 사이드바를 숨기면 화면을 넓게 사용할 수 있습니다.
- **`Ctrl + Tab`**: 열린 탭 사이 전환. 최근 사용한 파일 순서대로 정렬되어 있어 빠르게 이동할 수 있습니다.
- **`Cmd + Shift + E`**: 탐색기(Explorer) 패널로 이동. 파일 트리를 빠르게 확인할 수 있습니다.
- **`Cmd + G`**: 특정 라인으로 이동. 에러 메시지에 표시된 라인 번호로 바로 점프할 수 있습니다.

### 코드 편집 단축키

코드를 작성하고 수정하는 과정에서 가장 많이 사용하는 단축키들입니다.

- **`Cmd + D`**: 현재 선택한 단어와 같은 다음 단어를 추가 선택. 변수명을 한꺼번에 바꿀 때 매우 유용합니다.
- **`Cmd + Shift + L`**: 현재 선택한 단어와 같은 모든 단어를 한번에 선택. `Cmd + D`의 강화 버전으로 일괄 변경에 효과적입니다.
- **`Alt + Up/Down`**: 현재 줄을 위아래로 이동. 코드 순서를 바꿀 때 잘라내기/붙여넣기보다 훨씬 빠릅니다.
- **`Cmd + Shift + K`**: 현재 줄 삭제. 불필요한 코드를 빠르게 제거할 수 있습니다.
- **`Alt + Shift + Up/Down`**: 현재 줄 복제. 비슷한 코드를 반복적으로 작성할 때 시간을 절약합니다.
- **`Cmd + /`**: 주석 토글. 선택한 영역이나 현재 줄을 한번에 주석 처리/해제합니다.
- **`Cmd + Shift + [`** / **`]`**: 코드 접기/펼치기. 긴 파일에서 구조를 파악할 때 유용합니다.
- **`Cmd + Enter`**: 현재 줄 아래에 빈 줄 삽입. 커서 위치에 상관없이 다음 줄에서 작업을 시작할 수 있습니다.

### 검색과 바꾸기

프로젝트 전체에서 코드를 찾고 수정하는 작업은 일상적으로 발생합니다.

- **`Cmd + F`**: 현재 파일 내 검색.
- **`Cmd + H`**: 현재 파일 내 찾기 및 바꾸기.
- **`Cmd + Shift + F`**: 프로젝트 전체 검색. 특정 함수가 어디서 호출되는지 추적할 때 필수적입니다.
- **`Cmd + Shift + H`**: 프로젝트 전체에서 찾기 및 바꾸기. API 엔드포인트 변경 등 대규모 수정 작업에 사용합니다.

## 생산성을 높여주는 필수 확장 프로그램

VS Code의 진정한 강점은 풍부한 확장 프로그램 생태계에 있습니다. 수만 개의 확장 프로그램 중에서 검증된 필수 확장 프로그램들을 소개합니다.

### 코드 품질 관련

1. **ESLint**: JavaScript/TypeScript 코드의 문법 오류와 스타일 문제를 실시간으로 검사합니다. 팀 프로젝트에서 코드 일관성을 유지하는 데 필수적입니다.
2. **Prettier**: 코드 포매팅을 자동화합니다. 저장할 때 자동으로 코드 스타일을 정리해주므로 스타일 논쟁에서 해방될 수 있습니다.
3. **Error Lens**: 에러와 경고를 코드 옆에 인라인으로 표시합니다. Problems 패널을 열지 않아도 바로 문제를 확인할 수 있습니다.

### Git 관련

4. **GitLens**: Git 히스토리를 시각적으로 보여줍니다. 각 줄의 마지막 커밋 정보, blame 정보, 파일 히스토리 등을 에디터 안에서 바로 확인할 수 있습니다. 코드 리뷰나 버그 추적 시 매우 유용합니다.
5. **Git Graph**: Git 브랜치 구조를 그래프로 시각화합니다. 복잡한 브랜치 전략을 사용하는 프로젝트에서 현재 상태를 한눈에 파악할 수 있습니다.

### 개발 효율성

6. **Thunder Client**: VS Code 안에서 HTTP 요청을 보내고 응답을 확인할 수 있는 API 클라이언트입니다. Postman을 별도로 실행할 필요 없이 에디터 안에서 API 테스트가 가능합니다.
7. **Auto Rename Tag**: HTML/JSX에서 여는 태그를 수정하면 닫는 태그가 자동으로 함께 수정됩니다. React 개발 시 특히 편리합니다.
8. **Path Intellisense**: 파일 경로를 입력할 때 자동 완성을 제공합니다. import 구문 작성 시 오타를 방지할 수 있습니다.
9. **TODO Highlight**: 코드 내 TODO, FIXME 등의 주석을 시각적으로 강조합니다. 나중에 처리해야 할 작업을 놓치지 않도록 도와줍니다.

### 시각적 편의성

10. **Indent Rainbow**: 들여쓰기 레벨별로 색상을 다르게 표시합니다. Python이나 YAML 같이 들여쓰기가 중요한 언어에서 특히 유용합니다.
11. **Material Icon Theme**: 파일 확장자에 따라 아이콘을 표시합니다. 탐색기에서 파일 유형을 한눈에 구분할 수 있습니다.
12. **Bracket Pair Colorizer**: 중괄호, 괄호, 대괄호의 쌍을 색상으로 구분합니다. 중첩된 코드 구조를 파악하기 쉬워집니다. VS Code 내장 기능으로 대체되었지만 추가 기능이 있습니다.

## 최적의 settings.json 설정

VS Code의 설정 파일을 직접 편집하면 더 세밀한 커스터마이징이 가능합니다. `Cmd + Shift + P`로 명령 팔레트를 열고 "Open Settings (JSON)"을 검색하면 직접 편집할 수 있습니다.

```json
{
  "editor.fontSize": 14,
  "editor.lineHeight": 1.8,
  "editor.minimap.enabled": false,
  "editor.wordWrap": "on",
  "editor.bracketPairColorization.enabled": true,
  "editor.guides.bracketPairs": true,
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "editor.suggestSelection": "first",
  "editor.tabSize": 2,
  "editor.linkedEditing": true,
  "editor.cursorBlinking": "smooth",
  "editor.cursorSmoothCaretAnimation": "on",
  "editor.smoothScrolling": true,
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "files.trimTrailingWhitespace": true,
  "files.insertFinalNewline": true,
  "workbench.startupEditor": "none",
  "workbench.tree.indent": 16,
  "terminal.integrated.fontSize": 13,
  "explorer.confirmDelete": false,
  "explorer.confirmDragAndDrop": false
}
```

### 주요 설정 설명

- **`editor.formatOnSave`**: 파일을 저장할 때 자동으로 Prettier가 코드를 포매팅합니다. 수동으로 포매팅할 필요가 없어집니다.
- **`editor.codeActionsOnSave`**: 저장 시 ESLint가 자동으로 수정 가능한 문제를 고쳐줍니다.
- **`editor.linkedEditing`**: HTML 태그 편집 시 여는 태그와 닫는 태그가 동시에 수정됩니다.
- **`files.trimTrailingWhitespace`**: 저장 시 줄 끝의 불필요한 공백을 자동 제거합니다. 불필요한 Git diff를 방지합니다.
- **`files.insertFinalNewline`**: 파일 끝에 빈 줄을 자동 추가합니다. POSIX 표준을 준수하며 일부 도구의 경고를 방지합니다.

## 워크스페이스 활용 팁

### 멀티 루트 워크스페이스

여러 프로젝트를 하나의 VS Code 창에서 관리할 수 있습니다. `File > Add Folder to Workspace`로 폴더를 추가하면 됩니다. 프론트엔드와 백엔드를 동시에 작업할 때 특히 유용합니다.

### 프로필 기능

VS Code의 프로필 기능을 사용하면 프로젝트 유형별로 다른 설정과 확장 프로그램을 사용할 수 있습니다. 예를 들어 React 개발용 프로필, Python 개발용 프로필, 글쓰기용 프로필을 각각 만들어 빠르게 전환할 수 있습니다.

### 스니펫 커스터마이징

자주 반복하는 코드 패턴을 스니펫으로 등록해두면 시간을 크게 절약할 수 있습니다. `Cmd + Shift + P`에서 "Configure User Snippets"를 검색하면 직접 만들 수 있습니다.

```json
{
  "React Functional Component": {
    "prefix": "rfc",
    "body": [
      "export default function ${1:ComponentName}() {",
      "  return (",
      "    <div>",
      "      $0",
      "    </div>",
      "  );",
      "}"
    ]
  }
}
```

## 터미널 활용법

VS Code의 통합 터미널은 외부 터미널 앱을 사용하는 것보다 편리한 점이 많습니다.

- **`` Ctrl + ` ``**: 통합 터미널 토글
- **`Cmd + Shift + [`** / **`]`**: 터미널 탭 전환
- 터미널을 에디터 옆에 배치하면 코드와 터미널 출력을 동시에 볼 수 있습니다
- 여러 터미널을 분할해서 개발 서버, 테스트, Git 작업을 동시에 수행할 수 있습니다

## 디버깅 활용

VS Code의 내장 디버거는 `console.log`보다 훨씬 강력합니다.

- **F5**: 디버깅 시작
- **F9**: 브레이크포인트 토글
- **F10**: 스텝 오버 (함수 안으로 들어가지 않고 다음 줄로)
- **F11**: 스텝 인 (함수 안으로 들어가기)

`launch.json` 파일을 설정하면 Node.js, Chrome, Python 등 다양한 런타임에서 디버깅이 가능합니다. 변수 값 확인, 조건부 브레이크포인트, 워치 표현식 등 강력한 기능을 활용하면 버그 추적 시간을 크게 줄일 수 있습니다.

## 마무리

VS Code는 단순한 텍스트 에디터가 아니라 강력한 개발 환경입니다. 오늘 소개한 단축키와 설정들을 하나씩 적용해 보세요. 처음에는 익숙하지 않더라도 일주일만 꾸준히 사용하면 자연스럽게 손에 익게 됩니다.

가장 중요한 것은 자신의 워크플로우에 맞게 커스터마이징하는 것입니다. 모든 설정을 한번에 바꾸기보다는 하루에 하나씩 새로운 단축키를 익히는 것을 추천합니다. 작은 변화가 쌓이면 개발 생산성이 눈에 띄게 향상될 것입니다.
